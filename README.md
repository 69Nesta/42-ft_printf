# ğŸ“Œ ft_printf

`ft_printf` is a custom reimplementation of the standard C `printf` function.
It prints formatted text to the standard output (`stdout`) and supports several of the most commonly used conversion specifiers.

This project is part of the **42 curriculum**, following its constraints (usage of `write()`, manual type handling, no forbidden functions, etc.).

---

## âœ¨ Features

`ft_printf` supports the following format specifiers:

| Specifier   | Description                              |
| ----------- | ---------------------------------------- |
| `%c`        | Prints a single character                |
| `%s`        | Prints a string                          |
| `%p`        | Prints a pointer in hexadecimal format   |
| `%d` / `%i` | Prints a signed integer                  |
| `%u`        | Prints an unsigned integer               |
| `%x`        | Prints a number in lowercase hexadecimal |
| `%X`        | Prints a number in uppercase hexadecimal |
| `%%`        | Prints a literal `%`                     |

## ğŸ§  Internal Logic Overview

* `ft_printf` initializes `va_list` and passes the format string to `ft_processing`.
* `ft_processing` scans the string:

  * Prints characters until a `%` is found.
  * When encountering a `%`, it calls `ft_type_selector`.
* `ft_type_selector` dispatches the correct type handler (`c`, `s`, `p`, `d`, etc.).
* Each `ft_type_*` function:

  * Prints the formatted value
  * Returns:

    * The number of printed characters
    * The number of characters consumed from the format string
* `ft_printf` finally returns the total number of characters printed.
